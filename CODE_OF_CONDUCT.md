# Contributor Code of Conduct

## Our Pledge

In the interest of fostering an open and welcoming environment, we as contributors and maintainers pledge to make participation in our project and our community a harassment-free experience for everyone, regardless of age, body size, disability, ethnicity, gender identity and expression, level of experience, nationality, personal appearance, race, religion, or sexual identity and orientation.

## Our Standards

### Expected Behavior

- Using welcoming and inclusive language
- Being respectful of differing viewpoints and experiences
- Gracefully accepting constructive criticism
- Focusing on what is best for the educational community
- Showing empathy towards other community members
- Demonstrating patience with learners at different skill levels
- Maintaining academic integrity in all contributions
- Respecting user privacy and data confidentiality

### Unacceptable Behavior

- The use of sexualized language or imagery, and sexual attention or advances
- Trolling, insulting/derogatory comments, and personal or political attacks
- Public or private harassment
- Publishing others' private information, such as a physical or electronic address, without explicit permission
- Any conduct which could reasonably be considered inappropriate in an educational setting
- Misuse of user data or analytics information
- Academic dishonesty, including plagiarism or unauthorized collaboration
- Deliberate introduction of security vulnerabilities or backdoors

## Reporting Guidelines

## Enforcement Guidelines

Project maintainers are responsible for clarifying the standards of acceptable behavior and are expected to take appropriate and fair corrective action in response to any instances of unacceptable behavior.

Project maintainers have the right and responsibility to remove, edit, or reject comments, commits, code, wiki edits, issues, and other contributions that are not aligned to this Code of Conduct, or to ban temporarily or permanently any contributor for other behaviors that they deem inappropriate, threatening, offensive, or harmful.

## Community Guidelines

### Educational Environment

Astro Diary is an educational project for the academic year 2025–2026. We prioritize:

- Learning and skill development
- Knowledge sharing and mentorship
- Constructive feedback and code reviews
- Documentation that helps others understand the project
- Supporting fellow students in their learning journey

### Collaboration Standards

- All contributions should be properly documented
- Code should follow established coding standards
- Issues should be clearly described with reproduction steps when applicable
- Pull requests should have clear descriptions of changes made
- Review process should be constructive and educational

## Technical & Educational Guidelines

### Code Quality

- Write clean, readable, and maintainable code
- Include appropriate comments for complex logic
- Follow established coding conventions for the project
- Test your changes before submitting
- Ensure data privacy and security best practices

### Academic Integrity

- Attribute sources and references appropriately
- Do not copy code without proper attribution
- Be honest about your contributions and capabilities
- Respect intellectual property rights
- Follow academic honesty policies of Erasmus Hogeschool Brussel

## Project-Specific Guidelines

### Data Ethics and Privacy

Astro Diary handles personal diary entries and user analytics. Contributors must:

- Respect user privacy and confidentiality
- Implement proper data protection measures
- Ensure compliance with data protection regulations
- Handle sensitive user data with care
- Never access or share user data without proper authorization

### Diary Content Considerations

- Be mindful that diary entries may contain sensitive personal information
- Implement appropriate access controls for diary data
- Ensure sentiment analysis and text processing respect user privacy
- Store analytics data separately from personal diary content
- Provide appropriate data deletion and export capabilities

### Analytics and Tracking

- User event tracking should be transparent and privacy-conscious
- Analytics should be used for educational purposes only
- Avoid collecting unnecessary personal information
- Implement proper data retention policies
- Ensure users have control over their data
